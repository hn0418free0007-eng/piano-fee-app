# 環境変数・Secrets

## 目的

必要な設定名、用途、取得場所、設定場所だけを記録します。実際の値は禁止です。

最終更新日: 2026-07-17

## Streamlit Secrets

| 変数名 | 用途 | 取得場所 | 設定場所 |
|---|---|---|---|
| `app.mode` | `cloud` または `demo` の運用区分 | 運用方針 | Streamlit Community Cloud → App Settings → Secrets |
| `app.timezone` | Calendar検索と表示のタイムゾーン | 運用地域 | Streamlit Secrets |
| `app.public_url` | OAuth完了後に戻る本番アプリURL | Streamlit Community CloudのApp URL | Streamlit Secrets、Supabase Auth URL Configuration |
| `app.allowed_emails` | アプリ利用を許可するGoogleアカウント一覧 | 運用管理者 | Streamlit Secrets、同じ対象をSupabase `allowed_teachers` に登録 |
| `supabase.url` | Supabase ProjectのAPI URL | Supabase Dashboard → Project Settings → API | Streamlit Secrets |
| `supabase.anon_key` | ブラウザー／アプリ用Publishableまたはanon key | Supabase Dashboard → Project Settings → API | Streamlit Secrets |
| `google_calendar.calendar_id` | 読み取るCalendar ID | Google Calendar設定 | Streamlit Secrets |

## 移行時だけ使うPowerShell環境変数

| 変数名 | 用途 | 取得場所 | 設定場所 |
|---|---|---|---|
| `SUPABASE_URL` | SQLite移行先Project | Supabase Dashboard → Project Settings → API | 移行を実行する同じPowerShellセッション |
| `SUPABASE_SERVICE_ROLE_KEY` | RLSを越えて管理移行する最重要キー | Supabase Dashboard → Project Settings → API | 移行を実行する同じPowerShellセッションだけ |
| `PIANO_DB_PATH` | 移行元／ローカルSQLiteのパスを上書き | ローカルファイル配置 | 必要な場合だけPowerShell環境変数 |
| `PIANO_PKCE_DB_PATH` | PKCE一時DBの保存先をテスト等で上書き | 開発者が決定 | 通常は未設定。テスト時のみ |

## Supabase Dashboardへ直接設定する値

| 設定名 | 用途 | 取得場所 | 設定場所 |
|---|---|---|---|
| Google Client ID | Google OAuthクライアント識別 | Google Cloud Console → Google Auth Platform → Clients | Supabase Authentication → Providers → Google |
| Google Client Secret | Google OAuthの秘密情報 | Google Cloud Console → Google Auth Platform → Clients | Supabase Authentication → Providers → Google |
| Supabase Callback URL | GoogleからSupabaseへ戻るURL | Supabase Google Provider画面 | Google CloudのAuthorized redirect URIs |
| Site URL | Authの既定戻り先 | Streamlit本番アプリURL | Supabase Authentication → URL Configuration |
| Redirect URLs | アプリが指定できる戻り先 | Streamlit本番URL、必要時のみローカルURL | Supabase Authentication → URL Configuration |

## TOMLの形

実値を入れない構造例です。

```toml
[app]
mode = "cloud"
timezone = "Asia/Tokyo"
public_url = "本番アプリURL"
allowed_emails = ["許可するGoogleアカウント"]

[supabase]
url = "Supabase Project URL"
anon_key = "Publishableまたはanon key"

[google_calendar]
calendar_id = "primary"
```

## 禁止事項

- 実値入り `secrets.toml` をGitへ追加しない
- `.env` をGitへ追加しない
- service-role keyをStreamlitアプリへ設定しない
- Google Client Secretをコード、README、docsへ書かない
- Secretをコミット、Issue、チャット、スクリーンショットへ載せない
- 実値を確認するためにログ出力しない
