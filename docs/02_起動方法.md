# 起動方法

## 目的

数か月ぶりでも、WindowsとPowerShellを使ってローカル開発を再開できる手順です。

最終更新日: 2026-07-17

## 1. VSCodeでWorkspaceを開く

1. Visual Studio Codeを起動します。
2. `ファイル` → `フォルダーを開く` を選びます。
3. 次のRepositoryルートを開きます。

```text
C:\Users\hn041\Documents\Codex\piano-fee-app
```

4. Explorerに `local_web_app`、`docs`、`README.md` が表示されることを確認します。

別PCでは保存場所が異なります。GitHubからcloneした `piano-fee-app` フォルダーを開いてください。

## 2. Codexを起動する

VSCodeまたはCodexアプリでRepositoryルートをWorkspaceとして開きます。最初に次のように依頼します。

```text
まず docs/08_CHATGPT引継ぎ.md を読んでください。
次に git status とテスト結果を確認してください。
```

ChatGPTやClaude Codeを使う場合も、同じ引継ぎファイルから開始します。

## 3. PowerShellを開く

VSCodeの `ターミナル` → `新しいターミナル` を選び、PowerShellを開きます。現在地を確認します。

```powershell
pwd
git status
```

Repositoryルートでない場合は移動します。

```powershell
cd C:\Users\hn041\Documents\Codex\piano-fee-app
```

アプリのコマンドは `local_web_app` で実行します。

```powershell
cd local_web_app
```

## 4. 仮想環境を準備する

既存の仮想環境が使える場合:

```powershell
.\.venv\Scripts\Activate.ps1
python --version
python -m pip install -r requirements.txt
```

`.venv` がない場合:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

PowerShellでスクリプト実行が拒否された場合、現在のPowerShellだけ許可します。

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

`-Scope Process` はターミナルを閉じると元に戻ります。

## 5. テストを実行する

変更前と変更後に実行します。

```powershell
python -m compileall -q app.py database.py services pages tests
python -m pytest -q
python tests\demo_acceptance.py
```

## 6. Streamlitを起動する

```powershell
python -m streamlit run app.py
```

ブラウザーで次を開きます。

```text
http://127.0.0.1:8501
```

Secretsがなければデモ・ローカルモードで起動します。実値入り `secrets.toml` はGitへ追加しません。

終了するときはPowerShellで `Ctrl + C` を押します。

## 7. Git状態を確認する

Repositoryルートへ戻ります。

```powershell
cd ..
git status
git diff
```

意図しないDB、Secrets、バックアップが含まれていないことを確認します。

## 8. コミットする

変更ファイルを個別に追加する方法を推奨します。

```powershell
git add docs
git diff --cached
git commit -m "変更内容を表すメッセージ"
```

`git add .` を使う場合は、必ず直前に `git status` で対象を確認します。

## 9. GitHubへPushする

```powershell
git push origin main
git status
git branch -vv
```

`main` と `origin/main` が同じコミットであることを確認します。

## 10. Streamlitを再デプロイする

通常、Community Cloudは `main` のpushを検出して自動再デプロイします。

1. https://share.streamlit.io/ を開きます。
2. 対象アプリを選びます。
3. Deploy logsで新しいコミットが反映されたことを確認します。
4. 自動更新されない場合はアプリメニューからRebootします。
5. 本番URLでログイン、今日の受付、主要画面を確認します。

## PowerShellを使う主な場面

- 仮想環境の作成・有効化
- Python依存関係のインストール
- Streamlit起動
- pytest実行
- Git操作
- SQLite移行プレビュー
- 移行時だけ環境変数を現在セッションへ設定

PowerShell環境変数は同じターミナル内だけで利用します。別ターミナルへ自動では引き継がれません。
