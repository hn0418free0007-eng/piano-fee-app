# リリースノート

## このファイルの目的

このプロジェクトの変更履歴を、人が後から読んですぐ理解できる形で時系列に管理します。

GitHubのコミット履歴だけでは分かりにくい、追加機能、修正理由、仕様・運用変更、削除内容、既知の問題、次回予定をリリース単位で記録します。

プロジェクトの現在状態は `docs/04_PROJECT_STATUS.md`、開発再開手順と技術的な引継ぎは `docs/08_CHATGPT引継ぎ.md` を参照してください。

## 運用ルール

- 新しいリリースを「最新版」の一番上へ追加する
- 以前の最新版は「過去リリース」へ移動する
- コード変更だけでなく、仕様変更や運用変更も記録する
- リリース日を書く
- 関連するコミットIDとコミットメッセージを書く
- 何を変更したかだけでなく、変更理由を書く
- 追加、修正、変更、削除を分けて記録する
- 未解決の既知の問題を書く
- 次回予定を書く
- Secret、APIキー、token、実メールアドレスなどの秘密情報を書かない
- 作業終了時に `docs/04_PROJECT_STATUS.md` と `docs/08_CHATGPT引継ぎ.md` との矛盾がないか確認する
