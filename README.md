# ピアノ教室 レッスン料・発表会費管理システム

## ファイル

- `ピアノ教室_レッスン料管理.xlsx`：マクロなしで使用できる管理ブック
- `LessonFeeManagement.bas`：標準VBAモジュール
- `ThisWorkbook.cls`：起動時・終了時の確認処理
- `当月受付.cls`：受付を「済」にした際の日付自動入力
- `設計書.md`：列、数式、条件付き書式、保守方法

登録済みの氏名・金額はすべて架空のサンプルです。実際に使い始める前に「生徒マスタ」と「入金履歴」のサンプルを置き換えてください。

## マクロなしでの使い方

1. 「生徒マスタ」に生徒情報を登録します。生徒IDは重複させないでください。
2. 「当月受付」の対象年月と生徒IDを確認します。
3. 封筒を受け取ったら「受付」を `済` にし、受取額・受取日・支払方法を入力します。
4. 封筒へ受領印を押した直後に「印鑑」を `済` にします。
5. 入金履歴へ手入力で転記したら「売上反映」を `済` にします。
6. 「ダッシュボード」で未処理件数と未入金一覧を確認します。

行の色は、白＝受付前、黄＝印鑑待ち、緑＝履歴反映待ち、青＝反映済です。受付済みにもかかわらず受取額または受取日が空欄の場合は、赤が最優先で表示されます。

## VBAの取り込み手順

1. `ピアノ教室_レッスン料管理.xlsx` をExcelで開きます。
2. 「名前を付けて保存」で、ファイル形式を「Excel マクロ有効ブック（*.xlsm）」に変更します。
3. `Alt + F11` でVisual Basic Editorを開きます。
4. プロジェクト名を右クリックし、「ファイルのインポート」で `LessonFeeManagement.bas` を取り込みます。
5. 左側の `ThisWorkbook` をダブルクリックし、既存コードを削除して `ThisWorkbook.cls` の `Option Explicit` 以降を貼り付けます。
6. 左側の「Microsoft Excel Objects」にある「当月受付」シートをダブルクリックし、`当月受付.cls` の `Option Explicit` 以降を貼り付けます。
7. 保存してExcelを閉じ、再度開いたときに「コンテンツの有効化」を選びます。

`.cls` をそのままインポートすると、Excelが別のクラスモジュールとして扱うことがあります。そのため、手順5・6は対象オブジェクトへコード本文を貼り付ける方法を推奨します。

## VBA機能

- `当月受付表を作成`：在籍中の生徒から当月受付表を再作成
- `入金履歴へ反映`：受付済みデータを履歴に追加し、売上反映を `済` に変更
- `印鑑未押印を確認`：受付済み・印鑑未押印の生徒を警告
- `バックアップコピー作成`：ブックと同じ場所の `backup` フォルダへ時刻付きコピーを保存
- 当月受付シートのイベント：受付を `済` にしたとき、空欄の受取日に当日を入力

Excelの「開発」タブからボタンを配置し、上記のマクロを割り当てると、日常操作をボタン中心にできます。

## 推奨するボタン配置

「当月受付」に次のボタンを置くことを推奨します。

- 今月の受付表を作成
- 入金履歴へ反映
- 印鑑未押印を確認
- バックアップ

## 未検証事項

この環境ではExcelをバックグラウンド生成に使用しましたが、VBAはブックへ埋め込んでいません。したがって、VBAのインポート後のボタン操作、マクロセキュリティ、各PC固有の設定は実機で確認してください。マクロなしのブック構造、数式、テーブル、入力規則、条件付き書式は生成後に機械的に検査しています。

## バックアップと個人情報

- 日常利用前にOneDrive等の履歴管理または定期バックアップを設定してください。
- 生徒情報を含むため、共有範囲とファイル権限を制限してください。
- 月末には「入金履歴」をCSVで別保存すると、将来のPython・SQLite移行が容易です。
